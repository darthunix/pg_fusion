2026-03-11

- Executor FSM cleanup:
  - Removed `LogicalPlan -> Explain` transition in `executor/src/fsm.rs`.
  - `Explain` remains valid only in `PhysicalPlan` (after `Translate` builds a physical plan).

- Executor pgscan decomposition:
  - Split heap-page decode pipeline into testable modules:
    - `executor/src/scan/heap/visibility.rs` (visibility bitmap decode/checks)
    - `executor/src/scan/heap/page_iter.rs` (line pointer/page tuple iteration)
    - `executor/src/scan/heap/decode.rs` (tuple decode to Arrow RecordBatch)
  - `HeapScanStream::poll_next` now orchestrates decoding via `decode_block_to_batch(...)` instead of embedding all page/tuple logic inline.

- Executor pgscan API tightening:
  - `executor/src/lib.rs` switched to internal scan module boundary (`mod scan`), with heap implementation under `scan/heap`.
  - Heap scan naming aligned for future scan families:
    - `PgTableProvider` -> `HeapScanProvider`
    - `PgScanExec` -> `HeapScanExec`
    - `PgScanStream` -> `HeapScanStream`
    - `HeapBlock` -> `HeapPageBlock`
    - `count_scans`/`for_each_scan` -> `count_heap_scans`/`for_each_heap_scan`
  - Public visibility for scan internals reduced to crate/super scope (`pub(crate)`/`pub(super)`), including `ScanRegistry`, scan traversal helpers, and heap decode/visibility iterators.
  - Removed unused pgscan API surface (`ScanRegistry::close_and_clear`) and unused `HeapPageBlock` metadata fields not consumed by decode path.
