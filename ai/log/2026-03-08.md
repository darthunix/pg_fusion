2026-03-08

- Discussed buffer-pin architecture tradeoffs for `pg_fusion` scan path.

- Current safety constraint is valid:
  - We copy heap pages out of PostgreSQL shared buffers into our SHM slots.
  - This decouples worker processing lifetime from backend pin lifetime.
  - Without copying, backend death or early unpin can invalidate worker reads.

- Confirmed pin semantics / process ownership:
  - Buffer pins are refcounts and are backend-local in ownership/accounting.
  - `BufferIsPinned` / local pin checks are per-current-backend, not global ownership transfer.
  - `IncrBufferRefCount` only works if the same backend already holds a pin.
  - Therefore, worker cannot "adopt" backend pins.

- Confirmed threading concern:
  - PostgreSQL server code is generally not thread-safe.
  - Using a dedicated thread in worker to run pg/pgrx buffer APIs is high-risk.

- Evaluated "separate pin process" idea:
  - It does not solve ping-pong overhead by itself.
  - It centralizes scan feed and reduces scalability (single producer bottleneck).
  - It keeps IPC complexity while removing parallelism benefits.

- Direction chosen:
  - Keep pinning in backends (where snapshot/MVCC context naturally lives).
  - Keep page copy for correctness/isolation.
  - Remove lockstep protocol instead of removing copy.

- Target protocol shape (pg18-oriented):
  - Backend acts as producer with credit-based flow control.
  - Fill multiple page slots per wakeup (windowed prefetch), not one page per request.
  - Signal only on state transitions (`empty->non-empty`, `full->has-space`, `eof`) rather than per page.
  - Keep relation open for scan lifetime (avoid per-page open/close).
  - Use pg18 read pipeline (`read_stream_*`, `StartReadBuffers/WaitReadBuffers`) and configured `io_method` (`worker` or `io_uring`) for better in-flight I/O.

- Key conclusion:
  - Biggest practical win is batching + streaming flow control on top of safe copying.
  - Attempting zero-copy via cross-thread/cross-process pin tricks is fragile and unlikely to beat a well-batched copy pipeline.
